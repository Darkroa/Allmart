AllMart — Deployment & Environment Setup Guide
===============================================
Last updated: July 2026

This file covers four scenarios:
  A. Local development (your laptop)
  B. Cloudflare Pages (storefront) + Railway/Render (API)  ← recommended for Cloudflare
  C. Vercel (full-stack)
  D. Self-hosted VPS with nginx

It also explains:
  • Which Replit-specific files can be ignored outside Replit
  • How to swap Object Storage from Replit GCS to Cloudflare R2 / AWS S3
  • Database options (Neon, Supabase, Railway Postgres)
  • Email (SMTP / Resend) configuration

===========================================================================
REPLIT-SPECIFIC FILES — WHAT TO DO OUTSIDE REPLIT
===========================================================================

These files are only used by the Replit IDE and deploy system.
You can ignore (or delete) them in any other environment.

  .replit          — Replit IDE config: workflows shown in the sidebar,
                     port mappings, deployment target ("autoscale").
                     Has no meaning outside Replit — ignored by Node, nginx,
                     Docker, Vercel, Railway, Cloudflare, etc.

  .replitignore    — Exactly like a .dockerignore file: lists paths to
                     exclude from Replit's deployment image to keep it small.
                     Outside Replit, just use .gitignore / .dockerignore.

  replit.nix       — NixOS package list for the Replit shell environment
                     (currently only `pkgs.nginx`).
                     Outside Replit, install nginx (or your reverse proxy)
                     using your OS package manager or a Dockerfile.

OBJECT STORAGE — REPLIT vs OUTSIDE REPLIT
===========================================================================

Inside Replit the app uses Replit Object Storage, which is a Google Cloud
Storage bucket accessed through a local sidecar at http://127.0.0.1:1106.
This sidecar DOES NOT exist outside Replit.

The storage client lives at:
  artifacts/api-server/src/lib/objectStorage.ts

--- OPTION 1: Cloudflare R2 (recommended for Cloudflare deployments) ---

R2 is S3-compatible, has zero egress fees, and integrates directly with
Cloudflare Pages / Workers.

1. Create an R2 bucket in your Cloudflare dashboard.
2. Generate an R2 API token (Account → R2 → Manage R2 API Tokens).
3. Set these environment variables on your API server host:

     R2_ACCOUNT_ID=your-cloudflare-account-id
     R2_ACCESS_KEY_ID=your-r2-access-key-id
     R2_SECRET_ACCESS_KEY=your-r2-secret-access-key
     R2_BUCKET_NAME=allmart-uploads
     R2_PUBLIC_URL=https://pub-xxxx.r2.dev   # or your custom domain

4. Replace the GCS client in objectStorage.ts with the AWS SDK v3
   pointing to the R2 endpoint:

     import { S3Client, PutObjectCommand, GetObjectCommand } from "@aws-sdk/client-s3";
     import { getSignedUrl } from "@aws-sdk/s3-request-presigner";

     const s3 = new S3Client({
       region: "auto",
       endpoint: `https://${process.env.R2_ACCOUNT_ID}.r2.cloudflarestorage.com`,
       credentials: {
         accessKeyId:     process.env.R2_ACCESS_KEY_ID!,
         secretAccessKey: process.env.R2_SECRET_ACCESS_KEY!,
       },
     });

   The presigned-URL flow, public-object serving, and private-object routes
   in artifacts/api-server/src/routes/storage.ts stay exactly the same.
   Only the underlying client and the getObjectEntityUploadURL() method
   need to be updated to call PutObjectCommand via getSignedUrl().

--- OPTION 2: AWS S3 ---

Same as R2 above but use a standard S3 endpoint:

     endpoint: `https://s3.${region}.amazonaws.com`

Set:
     AWS_REGION=eu-west-1
     AWS_ACCESS_KEY_ID=...
     AWS_SECRET_ACCESS_KEY=...
     S3_BUCKET_NAME=allmart-uploads

--- OPTION 3: Backblaze B2 (cheapest) ---

B2 is S3-compatible. Use the same @aws-sdk/client-s3 approach:

     endpoint: `https://s3.${region}.backblazeb2.com`

--- OPTION 4: Local disk (dev only, no uploads in production) ---

Set these in your .env for local dev:
     PRIVATE_OBJECT_DIR=./local-uploads
     PUBLIC_OBJECT_SEARCH_PATHS=./local-uploads

And add a simple express.static() middleware to serve them.
Uploads will NOT persist across deploys — use this only for local testing.

A. LOCAL DEVELOPMENT (your laptop)
===========================================================================

PREREQUISITES
  • Node.js 24+         https://nodejs.org
  • pnpm 9+             npm install -g pnpm
  • PostgreSQL 15+      https://www.postgresql.org/download
    OR use a cloud DB (Neon free tier — https://neon.tech)

STEP 1 — Clone and install

  git clone https://github.com/your-org/allmart.git
  cd allmart
  pnpm install

STEP 2 — Create .env in the project root

  # Database
  DATABASE_URL=postgresql://youruser:yourpassword@localhost:5432/allmart

  # Session
  SESSION_SECRET=replace-with-a-long-random-string-32-chars-min

  # Email — use Gmail App Password (see §Email below)
  SMTP_HOST=smtp.gmail.com
  SMTP_PORT=587
  SMTP_USER=your@gmail.com
  SMTP_PASSWORD=your-gmail-app-password

  # Stripe
  STRIPE_SECRET_KEY=sk_test_...
  STRIPE_PUBLIC_KEY=pk_test_...

  # Object Storage — choose one option from §Storage above
  # For local dev, disable storage by leaving these unset
  # (upload endpoints will return 500, which is fine for local dev)

  # Public URL for email reset links
  APP_URL=http://localhost:3000

STEP 3 — Create DB and push schema

  createdb allmart                            # or create via pgAdmin / psql
  pnpm --filter @workspace/db run push        # creates all tables

STEP 4 — Seed admin user

  pnpm --filter @workspace/scripts run seed-admin

  Admin email:    admin@allmart.com
  Admin password: admin@allmart1234

STEP 5 — Start both servers

  # Terminal 1
  PORT=8080 BASE_PATH=/api pnpm --filter @workspace/api-server run dev

  # Terminal 2
  PORT=3000 BASE_PATH=/ pnpm --filter @workspace/storefront run dev

  Open http://localhost:3000

  The Vite dev server already proxies /api → http://localhost:8080
  so you do NOT need nginx locally.

B. CLOUDFLARE PAGES (storefront) + RAILWAY (API)
===========================================================================

The AllMart architecture has two separate services:
  • Storefront — a React SPA. Builds to static HTML/JS/CSS. Runs anywhere.
  • API server  — a Node.js Express app. NEEDS a server (not Cloudflare Workers).

Cloudflare Pages hosts the static storefront for free (unlimited requests).
Railway (or Render, Fly.io) hosts the Express API.

--- WHY NOT CLOUDFLARE WORKERS FOR THE API? ---

The Express 5 API uses Node.js APIs (pg, nodemailer, fs, crypto, etc.) that
are not available in the Cloudflare Workers runtime. Migrating the API to
Workers would require rewriting all 30+ route files to use the Workers fetch
API instead of Express. This is a large effort with no practical benefit.

Use Railway (free tier available) or Render for the API — they run Node.js
exactly as on your laptop.

--- STEP-BY-STEP ---

PART 1 — Deploy the API to Railway

  1. Sign up at https://railway.app (free tier: 500 hrs/month)
  2. New Project → Deploy from GitHub repo
  3. Select the allmart repository
  4. Set the Root Directory to: artifacts/api-server
     (or leave at root and set a custom start command — see below)
  5. Custom start command:
       pnpm install && pnpm --filter @workspace/api-server run build && \
       PORT=8080 BASE_PATH=/api node --enable-source-maps artifacts/api-server/dist/index.mjs
  6. Add environment variables (Settings → Variables):

       DATABASE_URL=postgresql://...   ← Railway can provision Postgres for you
       SESSION_SECRET=...
       SMTP_HOST=smtp.gmail.com
       SMTP_PORT=587
       SMTP_USER=your@gmail.com
       SMTP_PASSWORD=...
       STRIPE_SECRET_KEY=sk_live_...
       R2_ACCOUNT_ID=...              ← see §Storage for R2 setup
       R2_ACCESS_KEY_ID=...
       R2_SECRET_ACCESS_KEY=...
       R2_BUCKET_NAME=allmart-uploads
       R2_PUBLIC_URL=https://pub-xxxx.r2.dev
       APP_URL=https://allmart.pages.dev   ← your Cloudflare Pages URL

  7. Under Settings → Networking → Public Networking, click Generate Domain.
     Note the URL, e.g.:  https://allmart-api.up.railway.app

  8. Push DB schema on Railway:
       In the Railway shell (or via a one-off command):
       DATABASE_URL=<your-railway-db-url> pnpm --filter @workspace/db run push
       pnpm --filter @workspace/scripts run seed-admin

PART 2 — Build the storefront pointing at the Railway API

  The storefront calls /api/* — in production these must be absolute URLs
  pointing to your Railway API. Update the Vite config so production builds
  use the Railway URL:

  In artifacts/storefront/vite.config.ts, inside the server.proxy block,
  this is dev-only. For production, the storefront HTML is served from
  Cloudflare Pages and the browser calls the Railway API directly.

  You need to set the API base URL at build time. The storefront currently
  uses a Vite proxy for /api in dev. For production (Cloudflare Pages)
  the browser must call the full Railway URL.

  Option A — Environment variable (recommended):
    Add to the storefront build:
      VITE_API_BASE_URL=https://allmart-api.up.railway.app/api

    Then in the API client (lib/api-client-react), use:
      const base = import.meta.env.VITE_API_BASE_URL ?? "/api";

  Option B — Cloudflare Pages redirect rules:
    In Cloudflare Pages → Settings → Functions → Add redirect rule:
      /api/* → https://allmart-api.up.railway.app/api/* (proxy, status 200)
    This lets the storefront keep calling /api/* and Pages proxies it.
    No code change needed. This is the easiest approach.

PART 3 — Deploy the storefront to Cloudflare Pages

  1. Sign up / log in at https://pages.cloudflare.com
  2. Create Application → Pages → Connect to Git → select allmart repo
  3. Build settings:
       Framework preset:  None (custom)
       Build command:
         pnpm install && PORT=18539 BASE_PATH=/ pnpm --filter @workspace/storefront run build
       Build output directory:  artifacts/storefront/dist/public
  4. Environment variables (for the build only):
       NODE_VERSION=20
       VITE_API_BASE_URL=https://allmart-api.up.railway.app/api
       PORT=18539
       BASE_PATH=/
  5. Save and deploy. Cloudflare Pages gives you a URL like:
       https://allmart.pages.dev
     You can add a custom domain in Pages → Custom Domains.

PART 4 — CORS on the API

  When the storefront is on allmart.pages.dev and the API is on
  allmart-api.up.railway.app, the browser makes cross-origin requests.
  Add a CORS origin to the API server:

  In artifacts/api-server/src/index.ts (or wherever cors() is configured):

    import cors from "cors";
    app.use(cors({
      origin: [
        "https://allmart.pages.dev",
        "https://yourdomain.com",   // add custom domain here too
      ],
      credentials: true,           // required for cookie-based auth
    }));

  Also update the session cookie config — set sameSite: "none", secure: true
  so cookies cross the Pages ↔ Railway boundary.

--- CLOUDFLARE R2 for uploads ---

  With the storefront on Cloudflare Pages and uploads going through the
  Railway API, file uploads work like this:

  1. Browser calls POST /api/storage/uploads/request-url (Railway API)
  2. Railway API generates a presigned R2 URL and returns it
  3. Browser uploads the file directly to R2 (bypasses your API server)
  4. Browser saves the objectPath reference in the product/order record

  This is the most efficient architecture — your API never streams large files.
  Set up R2 as described in §Storage above.

C. VERCEL (full-stack)
===========================================================================

Vercel can host BOTH the static storefront AND the Express API as a
serverless function — but Express serverless functions have cold starts and
are limited to 10 MB payload and 10 s timeout on the free plan.

For AllMart (large Express bundle ~3.3 MB), Vercel works but is better
suited for lower-traffic apps.

STEP 1 — vercel.json in the project root

  Create vercel.json:

  {
    "version": 2,
    "builds": [
      {
        "src": "artifacts/api-server/dist/index.mjs",
        "use": "@vercel/node"
      },
      {
        "src": "artifacts/storefront/dist/public/**",
        "use": "@vercel/static"
      }
    ],
    "routes": [
      { "src": "/api/(.*)", "dest": "artifacts/api-server/dist/index.mjs" },
      { "src": "/(.*)",     "dest": "artifacts/storefront/dist/public/$1" }
    ]
  }

STEP 2 — Build before deploying

  PORT=18539 BASE_PATH=/ pnpm --filter @workspace/storefront run build
  pnpm --filter @workspace/api-server run build

STEP 3 — Set env vars in Vercel dashboard

  Same list as Railway above (DATABASE_URL, SESSION_SECRET, SMTP_*, etc.)
  Storage: use Vercel Blob (free tier) or Cloudflare R2 (recommended).

STEP 4 — Deploy

  npm install -g vercel
  vercel --prod

  Vercel auto-detects the vercel.json and deploys both services.

STEP 5 — DB schema

  DATABASE_URL=<vercel-postgres-url> pnpm --filter @workspace/db run push

  Note: Vercel Postgres (Neon-backed) is free up to 256 MB.

D. SELF-HOSTED VPS (Ubuntu / Debian with nginx)
===========================================================================

Use this if you have a DigitalOcean Droplet, Hetzner VPS, or similar.

STEP 1 — Server setup

  sudo apt update && sudo apt install -y nginx postgresql nodejs npm
  npm install -g pnpm

STEP 2 — Clone and install

  git clone https://github.com/your-org/allmart.git /opt/allmart
  cd /opt/allmart
  pnpm install

STEP 3 — .env file

  cp .env.example .env   # or create manually — see §A STEP 2 above
  nano .env

STEP 4 — DB setup

  sudo -u postgres createdb allmart
  sudo -u postgres psql -c "CREATE USER allmart WITH PASSWORD 'yourpassword';"
  sudo -u postgres psql -c "GRANT ALL ON DATABASE allmart TO allmart;"
  pnpm --filter @workspace/db run push
  pnpm --filter @workspace/scripts run seed-admin

STEP 5 — Build

  PORT=18539 BASE_PATH=/ pnpm --filter @workspace/storefront run build
  pnpm --filter @workspace/api-server run build

STEP 6 — systemd service for the API

  Create /etc/systemd/system/allmart-api.service:

  [Unit]
  Description=AllMart API Server
  After=network.target postgresql.service

  [Service]
  Type=simple
  User=www-data
  WorkingDirectory=/opt/allmart
  EnvironmentFile=/opt/allmart/.env
  Environment=PORT=8080
  Environment=BASE_PATH=/api
  ExecStart=/usr/bin/node --enable-source-maps artifacts/api-server/dist/index.mjs
  Restart=always
  RestartSec=5

  [Install]
  WantedBy=multi-user.target

  sudo systemctl enable allmart-api
  sudo systemctl start allmart-api

STEP 7 — nginx config

  Replace /etc/nginx/sites-available/default with:

  server {
    listen 80;
    server_name yourdomain.com;

    # Serve React SPA
    root /opt/allmart/artifacts/storefront/dist/public;
    index index.html;

    location /api/ {
      proxy_pass http://127.0.0.1:8080;
      proxy_http_version 1.1;
      proxy_set_header Host $host;
      proxy_set_header X-Real-IP $remote_addr;
      proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
      proxy_set_header X-Forwarded-Proto $scheme;
      proxy_read_timeout 60s;
      client_max_body_size 20M;
    }

    location / {
      try_files $uri $uri/ /index.html;  # SPA fallback
    }
  }

  sudo nginx -t && sudo systemctl reload nginx

STEP 8 — HTTPS (Let's Encrypt)

  sudo apt install -y certbot python3-certbot-nginx
  sudo certbot --nginx -d yourdomain.com

  Certbot auto-renews. Done.

EMAIL SETUP (SMTP)
===========================================================================

AllMart uses SMTP as the primary email transport with Resend as fallback.

--- Gmail (recommended for low volume) ---

  1. Enable 2FA on your Gmail account.
  2. Go to https://myaccount.google.com/apppasswords
  3. Create an App Password for "Mail" / "Other (Custom name)".
  4. Use the 16-character password as SMTP_PASSWORD.

  SMTP_HOST=smtp.gmail.com
  SMTP_PORT=587
  SMTP_USER=your@gmail.com
  SMTP_PASSWORD=xxxx xxxx xxxx xxxx   (no spaces in the actual value)

  Limit: 500 emails/day free.

--- Zoho Mail (better for custom domain) ---

  SMTP_HOST=smtp.zoho.com
  SMTP_PORT=587
  SMTP_USER=admin@yourdomain.com
  SMTP_PASSWORD=your-zoho-password

--- Resend (fallback / high volume) ---

  Get a free API key at https://resend.com (3,000 emails/month free).
  Set RESEND_API_KEY in your env. Resend is used automatically if SMTP fails.

TELEGRAM BOT
===========================================================================

  1. Create a bot via @BotFather on Telegram → /newbot
  2. Copy the token → TELEGRAM_BOT_TOKEN
  3. Get your chat ID → message @userinfobot or @getidsbot
  4. Set TELEGRAM_CHAT_ID and TELEGRAM_WEBHOOK_SECRET (any random string)
  5. Deploy the app so it's reachable over HTTPS.
  6. Register the webhook:
       curl -X POST https://api.telegram.org/bot<TOKEN>/setWebhook \
         -d url=https://yourdomain.com/api/telegram/webhook \
         -d secret_token=<TELEGRAM_WEBHOOK_SECRET>
     Or use Admin → Telegram → Register webhook in the UI.

Bot commands:
  /orders      — last 5 orders
  /pending     — orders awaiting verification
  /notifications — latest admin notifications
  /help        — command list
