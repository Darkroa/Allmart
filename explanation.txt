NowBuy — Setup, Storage & Database Guide
=========================================

## 1. Object Storage (local development)

NowBuy uses Replit Object Storage to host uploaded images (payment screenshots,
product images, etc.).  Three environment variables control it:

  DEFAULT_OBJECT_STORAGE_BUCKET_ID   — the bucket ID assigned by Replit
  PRIVATE_OBJECT_DIR                 — internal prefix for private files
  PUBLIC_OBJECT_SEARCH_PATHS         — comma-separated prefix list for public serving

To run locally (outside Replit) you need a Postgres DB and your own file storage:

  1. Copy `.env.example` (or create a `.env`) with:

       DATABASE_URL=postgres://user:pass@localhost:5432/nowbuy
       SESSION_SECRET=any-random-string-32-chars-min

  2. For images locally, you can swap the `useImageUpload` hook to upload to any
     S3-compatible service.  The hook lives at:

       artifacts/storefront/src/hooks/use-image-upload.ts

  3. Install dependencies:

       pnpm install

  4. Push the DB schema:

       pnpm --filter @workspace/db run push

  5. Seed the admin user:

       pnpm --filter @workspace/scripts run seed-admin

  6. Start the dev server:

       pnpm --filter @workspace/api-server run dev      # port 8080
       pnpm --filter @workspace/storefront run dev       # port 18539


## 2. Telegram Bot Setup

You need three secrets set in Replit (Settings → Secrets):

  TELEGRAM_BOT_TOKEN      — get from @BotFather (e.g. 7123456789:AAF...)
  TELEGRAM_CHAT_ID        — your personal or group chat ID (e.g. -100123456789)
  TELEGRAM_WEBHOOK_SECRET — any random string you choose (e.g. mysecret123)
  TELEGRAM_BOT_NAME       — your bot's @username (e.g. @MyShopBot) — optional,
                            used in command matching (e.g. /orders@MyShopBot)

After setting secrets:
  1. Deploy or run the app so it's reachable over HTTPS.
  2. Go to Admin → Telegram bot → click "Register webhook".
  3. Telegram will now push all bot messages to your server automatically.

Bot commands:
  /orders         — last 5 orders
  /pending        — orders awaiting payment screenshot verification
  /notifications  — latest admin notifications
  /help or /start — command reference


## 3. Hourly Automatic Database Backup

A backup script lives at:  scripts/src/backup.ts

It dumps all users (without password hashes), products, and orders to a JSON file.

### Run manually

  pnpm --filter @workspace/scripts run backup

The output file is saved in:  ./backups/backup-<timestamp>.json

### Schedule hourly backups (Linux / Replit shell)

Add a cron job in the Replit shell:

  crontab -e

Then add this line (backs up every hour):

  0 * * * * cd /home/runner/workspace && pnpm --filter @workspace/scripts run backup >> /tmp/backup.log 2>&1

To verify cron is running:

  tail -f /tmp/backup.log


## 4. DB Schema Push (dev → prod / new DB)

To apply the latest schema to any database:

  DATABASE_URL=postgres://user:pass@host/dbname pnpm --filter @workspace/db run push

This uses Drizzle's push command which introspects the target DB and applies only
the missing columns / tables without destroying existing data.

WARNING: drizzle push is safe for additive changes (new columns, new tables).
For destructive changes (dropping columns) always back up first.


## 5. Rolling Back the DB by 1 Hour

Replit's built-in Postgres does not expose point-in-time recovery.
To "roll back" 1 hour:

  1. Restore from the last hourly backup:

       node -e "
         const b = require('./backups/backup-<timestamp>.json');
         console.log('Products:', b.products.length, 'Orders:', b.orders.length);
       "

  2. Use the backup JSON as a seed file:
     Write a restore script at scripts/src/restore.ts that reads the JSON and
     re-inserts rows (using INSERT … ON CONFLICT DO UPDATE).

  3. For a full Postgres restore from pg_dump (if you use a self-hosted DB):

       pg_restore -d nowbuy -U postgres backup.dump


## 6. Pushing Dev DB to a New/Production DB

  1. Take a backup:

       pnpm --filter @workspace/scripts run backup

  2. Export with pg_dump (if on self-hosted Postgres):

       pg_dump $DATABASE_URL -Fc -f backup.dump

  3. Push schema to new DB:

       DATABASE_URL=<new-db-url> pnpm --filter @workspace/db run push

  4. Import data:

       pg_restore --no-owner -d <new-db-url> backup.dump

  5. Run seed-admin to restore the admin user if needed:

       DATABASE_URL=<new-db-url> pnpm --filter @workspace/scripts run seed-admin
